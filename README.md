I worked through this project at the system level and took overarchingly big approaches to solving smaller problems. My main technical goal was to develop an efficient way to store, organize, and retrieve course data for an academic advising system. Specifically, my program had to read course data from a file, display the courses in alphanumeric order, and find and display a course based on the course number provided and any desired prerequisites given. These requirements highlighted the general but important differentiation that must be made between simply needing to store this type of data versus needing to store it in a way that best fits the structure of the data itself and the program’s required uses of it .

In order to find the best suited data structure, vectors, hash tables, and binary trees were compared and contrasted. This was a lesson in how general differences such as simplicity, search performance, and memory use can drastically alter the application as a whole (sorting and linear searches for vectors, fast search speed for tables with the loss of natural order, etc.).  After observing and testing the three data structures, it was found that the binary tree is optimal in fulfilling the basic requirements of quickly finding the specified course and then printing all in order .

One large obstacle of actually implementing the program was prerequisite validation, most importantly making sure that the provided prerequisite(s) (if any) existed amongst the stored courses. To overcome this, I used a separate two-pass read approach where the first pass simply saved all the courses in the tree before validating if given prerequisites are in the set of course numbers . 

This project taught me to start focusing on what I’m going to do before I start typing. It’s more important to write some pseudocode to work out how the program is going to operate and analyze how efficient the operations will be and scale before you even begin to write the statements. I have started to organize my work into separable parts and written pseudocode accordingly for each separated functionality (file I/O, building the tree data structure, user request handling).
